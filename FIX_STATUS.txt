🐛 PROBLEM FOUND:
   The datetime_with_tz pattern wasn't capturing the date portion properly.
   It used non-capturing groups (?:...) so the date was hidden in match[0].
   This caused "NO DETECTION" errors in 46% of tests.

🔧 FIX APPLIED:
   ✅ Changed pattern to capture date in group 1
   ✅ Simplified parseMatch() to use direct groups
   ✅ Expanded separator support (-, /, .)
   
   Pattern now captures:
   • match[1] = date (e.g., "2025-07-18")
   • match[2] = time (e.g., "18:15:23")
   • match[3] = timezone (e.g., "EST")
